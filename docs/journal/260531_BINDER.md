# DML Binder

It's time to think about SQL expression execution model, which involves
connecting the AST and the table values. This is kinda tricky because there are
four different things I need to connect:

- AST data type
- Table data type
- Row data type
- Runtime data type

Okay, more like three, because rows can directly return the runtime data type
that the evaluator uses. Still, there's some concerns before moving on to
implementing the binder, which itself is a question too.

- What even is a runtime data type?
- Can rows or functions return dynamic data type?
- How should a "resolved AST" and dependencies be expressed on the data model?

Also, there's some thought in the corner yelling to make a bytecode, going
"`and $0, $1; ret $0` is so beautiful, isn't it?". Some idea is valid, but
I'd rather have a recursive evaluator because otherwise I end up creating
one more representation model when I really want to have less.

## Runtime data type

But first of all, I need to figure out what "runtime data type" means first.
It'll be the basis of all runtime engine, ideally the format the row iterator
uses, which means the evaluator would also use. Currently, the row is expressed
as `std::vector<std::variant<...>>`, which works, but it may be not an ideal
format.

But then again, I can't think of a better one. This is effectively equivalent
to creating a programming language - it literally is - and the need is pretty
much equivalent to making bunch of classes going `IntegerValue`, `StringValue`,
etc, that can hold the value and additional useful information.

There are probably three approaches to do this in C++, one is going full OOP
and virtual classes - which may not be tolerable for high-performance
applications, which is the case for databases, but it's negligible for now.
One is just using `std::variant` like it's done right now.
One is COM VARIANT sadness that gives you unions and `void*`.

I think a good compromise is just keep using `std::variant` as it is preferred
for modern C++ development, but using a wrapper struct to give it a structure
to contain additional information should I have them.

For instance, something like this:

```cpp
struct NullValue {};

struct IntegerValue {
  std::int64_t value;
};

using Value = std::variant<IntegerValue, NullValue>;
```
